Overview

This charting library simplifies creating multiple instances for each of the 50 states. The user gives data for all 50 states, a graph() function, a regraph() function and the library will take care of the rest.

The minimal code to create a 50graph chart is as follows:


	var data = [{
		"data": { ... },
		"label": "AK"
	},{
		"data": { ... },
		"label": "ME"
	}, {
		"data": { ... },
		"label": "VT"
	}, ...
	];

	var config = {
		"el": "#g-container",
		"data": data,
		"break": 768
	};

	var map = new Fifty_graphs(config);

	map.graph(state_data, g, vars){
		/* Code for each state graph goes here */
	}

	map.draw()

Draw and redraw

The api creates 50 svg g elements and places them in the appropriate layout. The graph() and regraph() functions will be called for each g element and provided three parameters: state_data, g, vars.

Note that the graph() and regraph() function will have (0,0) be the top left corner of that particular svg g element. 


	state_data: the individual "data" that matches that state's label.
	g: the d3 selection for that svg g element.
	vars: the JS object that contains 4 variables.
		vars.svg_width: the width of the svg
		vars.svg_height: the height of the svg
		vars.g_width: the width of the g element's alloted space.
		vars.g_height: the height of the g element's alloted space.

Data Format

Data must be in the form of an array of JSON objects with a "label" key with the state's abbreviation and a "data" key with whatever data you will need to access.

Note: You will be able to have this data be editable and repass into the draw function.


	[{
		"data": { ... },
		"label": "AK"
	},{
		"data": { ... },
		"label": "ME"
	}, {
		"data": { ... },
		"label": "VT"
	}, ...

Options

There are a few options avaialable for the map


	var config = {
		el: [string] the div id that you are appending the graphic to,
		data: [object] the data you are supplying to the initial graph() function,
		breakpoint: [int] the pixel width that you want the graphic to rotate 90 degrees
	}

Update Data

For more interactive work, you may want to allow the state's data to update and have 50graphs redraw itself. There is a function to update the data before calling redraw(). 


	/* Update Data however you would like */
	map.updateData(data);
	map.redraw();
